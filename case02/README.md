#목표
    서버가 실행 후 계속 대기하고
    클라이언트가 다중으로 붙을 수 있는 형태 구현

#작동
## 서버
    1. 서버가 클라이언트 대기
    2. 클라이언트 접속
    3. 메시지 수신 대기
    4. 메시지 수신시 출력

## 클라이언트
    1. 서버 접속
    2. 서버에 메시지 전송 > 메시지는 콘솔에서 전송가능 하도록함.
    3. 전송 완료 후 다시 메시지 전송
